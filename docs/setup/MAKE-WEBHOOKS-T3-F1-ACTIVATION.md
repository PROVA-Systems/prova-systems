## T3 — Termin-Reminder

### Zweck
Sendet automatische Termin-Erinnerungs-Emails an SVs (Tag-1-vor-Termin, Tag-3-vor-Termin, etc.).

### Aktivierung (10 Min)

1. **Make.com Login** → https://eu1.make.com → Marcel-Account
2. **Scenario öffnen:** Scenario-ID `5147519` aufrufen oder via URL: `https://eu1.make.com/scenario/5147519`
3. **Gmail-Connection einrichten:**
   - Bei der Gmail-Action im Scenario auf "Add a connection"
   - "Sign in with Google" → Marcel's `kontakt@prova-systems.de` Account
   - OAuth-Permission "Send mail on your behalf" zustimmen
   - Verbindung testen via "Re-validate"
4. **Test-Run:**
   - "Run once"-Button klicken
   - Test-Daten verwenden (Datum-in-Zukunft, eigene Email als Empfänger)
   - Verify: Email kommt an
5. **Aktivieren:**
   - "ON"-Toggle oben rechts klicken
   - Schedule verifizieren: täglich oder pro-Termin-Trigger?
   - Save

### Verification
```bash

curl -X POST https://hook.eu1.make.com/<HOOK-ID-T3> \
  -H "Content-Type: application/json" \
  -d '{"sv_email":"test@prova-systems.de","termin_datum":"2026-05-20","termin_typ":"Ortstermin","aktenzeichen":"SCH-2026-001","tage_bis":1}'

# Erwartung: 200 OK + Test-Email in Marcel-Inbox
```

## Troubleshooting

### Gmail-OAuth fehlgeschlagen
- Marcel's Google-Account hat 2FA — App-Passwort erstellen unter https://myaccount.google.com/apppasswords
- Alternativ: Service-Account mit Domain-Wide-Delegation (Enterprise-Setup)

### Stripe-Webhook-Trigger feuert nicht
- Stripe-Dashboard → Webhooks → Make-URL geprüft
- Event-Type: muss `customer.subscription.created` enthalten
- Test-Webhook senden via Stripe-CLI oder Dashboard-Button

### Make-Scenario stoppt bei Errors
- Make-Scenario-Settings → "Sequential processing" → "Allow storing incomplete executions"
- Für Failure-Recovery + Marcel-Review
