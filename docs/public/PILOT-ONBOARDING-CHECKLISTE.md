# PROVA Pilot-Onboarding-Checkliste

**Stand:** 02.05.2026 · Skeleton — wird in Sprint S6 Phase 5 erweitert.

---

## Wofür diese Datei?

Schritt-für-Schritt-Checkliste für Pilot-SVs. Auch als interne Onboarding-Vorlage für Marcel.

---

## Vor dem ersten Call (Marcel-Vorbereitung)

- [ ] Pilot-Vereinbarung (`PILOT-VEREINBARUNG-ENTWURF.md`) als PDF + AVV als Anlage
- [ ] Demo-Fall `SCH-DEMO-001` vorbereitet
- [ ] Onboarding-Webinar-Slot kalendert
- [ ] Pilot-Stripe-Coupon `FOUNDING-99` aktiv

---

## Onboarding-Call (60 Minuten)

### Teil 1 — Begrüßung + Erwartungen (10 Min)
- [ ] Vorstellung Marcel + PROVA-Vision
- [ ] Pilot-Programm-Eckpunkte erläutern
- [ ] Pilot-Vereinbarung + AVV durchgehen, signieren

### Teil 2 — Account-Anlage (10 Min)
- [ ] Account auf `app.prova-systems.de/register` anlegen
- [ ] E-Mail-Bestätigung
- [ ] 2FA aktivieren (TBD wenn implementiert, sonst späteres Sprint)
- [ ] Workspace „Solo" wählen (Founding-Member-Coupon angewandt)

### Teil 3 — Stamm-Daten (15 Min)
- [ ] Profil-Page: SV-Name, IHK-Nr, Spezialisierung
- [ ] Briefkopf-Daten (`profil-supabase.html`)
- [ ] Erste 3 Auftraggeber/Kontakte anlegen
- [ ] Bank-Daten für Rechnungen

### Teil 4 — Demo-Fall (15 Min)
- [ ] Demo-Fall `SCH-DEMO-001` automatisch geladen (TBD wenn Sprint 20 implementiert)
- [ ] Akte öffnen, durchsehen
- [ ] §6-Editor demonstrieren
- [ ] PDF-Export ausprobieren

### Teil 5 — Erste echte Probe-Akte (10 Min)
- [ ] Pilot-Teilnehmer wählt einen leichten echten Fall (oder Test-Fall)
- [ ] Marcel begleitet erste Akten-Anlage
- [ ] Diktat aufnehmen oder Text eingeben
- [ ] §§-Generator durchlaufen
- [ ] Speichern

### Teil 6 — Termine + Feedback-Plan (5 Min)
- [ ] Wöchentliche 30-Min-Calls vereinbaren (6 Wochen)
- [ ] Mid-Pilot-Review (Woche 4) und End-of-Pilot-Review (Woche 6) terminieren
- [ ] Feedback-Kanal klären (E-Mail, Slack, WhatsApp?)

---

## Erste Woche (Pilot-Selbstständigkeit)

- [ ] 1-2 echte Gutachten-Fälle anlegen
- [ ] §6-Editor mindestens 1× nutzen
- [ ] PDF-Export mindestens 1× nutzen
- [ ] Bei Bug/Frage: E-Mail an Marcel oder Slack

---

## Wöchentliche Check-Calls (30 Min)

- Was funktioniert?
- Was nervt?
- Was fehlt?
- Bug-Liste durchgehen
- Roadmap-Sneak-Peek aus PROVA-VISION-MASTER.md

---

## Mid-Pilot-Review (Woche 4)

- [ ] Schriftliches Feedback (5-10 Punkte)
- [ ] Net Promoter Score (NPS) erfragen
- [ ] Erste Referenz-Bereitschaft klären

---

## End-of-Pilot-Review (Woche 6)

- [ ] Schriftliches Final-Feedback
- [ ] Subscription-Entscheidung (Solo / Team / Founding-99€)
- [ ] Referenz-Einwilligung (anonym / mit Logo / voll)
- [ ] Empfehlung an Kollegen?
- [ ] Falls Abbruch: Daten-Export anbieten + Lessons-Learned-Call
